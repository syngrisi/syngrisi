## Цели
- Добавить отдельное представление `/baselines` с таблицей бейзлайнов, превью изображений, сортировкой/фильтрами/пагинацией и счетчиком использования в чеках.
- Клик по строке/превью ведет в основной чек-вью, отфильтрованный по использованию выбранного бейзлайна.
- Реализация не должна ломать существующие ручки и должна быть изолированной от новой админ-задачи по очистке осиротевших бейзлайнов.

## Данные и сервер
- Бейзлайны остаются на существующей ручке `GET /v1/baselines`, но добавляется опциональный флаг `includeUsage` (string bool). При его наличии контроллер извлекает идентификаторы `snapshootId` из текущей страницы, агрегирует количество чеков `Check.baselineId` по этим id и возвращает поле `usageCount` в каждом элементе. В остальных случаях ручка ведет себя как раньше.
- Для превью фронт передает `populate=snapshootId`, чтобы получить `filename` из снимка без дополнительных запросов.
- В `GET /v1/tests` добавляется мягкий фильтр `baselineSnapshotId`: перед пагинацией тестов сервис подгружает id тестов у чеков с `baselineId` равным переданному id и сужает фильтр `_id: { $in: [...] }`. При пустом списке возвращается пустая выдача. Опция никак не влияет на остальные фильтры/сортировки.
- Логика подсчета использования и фильтра по бейзлайну оформлена в отдельных вспомогательных функциях, чтобы их можно было покрыть unit-тестами без реальной БД.

## Фронтенд
- Новый маршрут `/baselines` в приложении index2 с использованием общего QueryParamProvider для синхронизации фильтров/сортировки/страницы в URL.
- Страница состоит из:
  - панели фильтров (name/branch/browser/os/viewport/app как строковые фильтры, поиск по подстроке через `$regex`/`$options: 'i'`) и сортировки (по createdDate/name/branch/updatedDate; `usageCount` исключен из сортировки),
  - таблицы Mantine `Table` в `ScrollArea` с выбором видимых колонок (`MultiSelect` с хранением в localStorage). Колонки по умолчанию: превью, name, branch, browserName, viewport, os, createdDate, usageCount, markedAs.
  - в ячейке превью миниатюра (80–100px) и `Tooltip`/`HoverCard` с увеличенным изображением (300–400px).
  - строка кликабельна и ведет на `/` c установкой query-param `filter={"baselineSnapshotId":"<snapshootId>"}` для чек-вью.
  - пагинация Mantine `Pagination` и `Select` для выбора лимита, бэкенд-пагинация через `page/limit`.
- Состояние загрузки/ошибок отображается через Mantine `Loader`/`Alert`. Счетчик использования подписан как справочный и не участвует в сортировке.

## Навигация и маршрутизация
- В глобальной навигации добавляется пункт “Baselines” (группа main) с переходом на `/baselines`.
- Страница использует общие хедер/навигацию IndexLayout, но не зависит от текущей таблицы тестов.
- Редирект в чек-вью использует существующую страницу `/` (Tests) и только модифицирует query params, не трогая состояние базового списка.

## Независимость задач
- Бэк и фронт части baselines-вью общаются только через расширенную ручку `/v1/baselines` и дополнительный фильтр `baselineSnapshotId` в `/v1/tests`.
- Админ-задача очистки осиротевших бейзлайнов реализуется отдельной ручкой `/v1/tasks/task_handle_orphan_baselines` и не пересекается с logics `/baselines`.
- Документация и планы вынесены в отдельные файлы, чтобы можно было реализовывать/тестировать части независимо.
