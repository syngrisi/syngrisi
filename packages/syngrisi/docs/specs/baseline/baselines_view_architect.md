## Цели
- Добавить отдельное представление `/baselines` с таблицей бейзлайнов, превью изображений, бесконечной прокруткой поверх бэкенд-пагинации, сортировкой/фильтрами/переключаемыми колонками и счетчиком использования в чеках.
- Клик по строке/превью ведет в основной чек-вью, отфильтрованный по использованию выбранного бейзлайна.
- Реализация не должна ломать существующие ручки и должна быть изолированной от новой админ-задачи по очистке осиротевших бейзлайнов.

## Данные и сервер
- Бейзлайны остаются на ручке `GET /v1/baselines` с опциональным флагом `includeUsage` (string bool). При его наличии контроллер собирает `snapshootId` текущей страницы (учитывая populate), агрегирует `Check.baselineId` и добавляет поле `usageCount` в каждый элемент. Без флага ответ как раньше.
- Для превью фронт передает `populate=snapshootId`, чтобы получить `filename` без дополнительных запросов.
- В `GET /v1/tests` добавлен фильтр `baselineSnapshotId`: сервис получает id тестов по `Check.baselineId`, сужает выборку `_id: { $in: [...] }`, при отсутствии совпадений возвращает пустую страницу.
- Контроллер `/v1/baselines` работает через `ensureLoggedInOrApiKey`; для роли `user` фильтр дополняется `markedByUsername=req.user.username`.

## Фронтенд
- Новый маршрут `/baselines` в index2 (QueryParamProvider); основные параметры (`filter`, `sortBy`, `app`, `base_filter`) синхронизируются через use-query-params.
- Страница состоит из:
  - двух выдвижных панелей: сортировка (поля name/branch/createdDate/browserName/viewport/os/markedAs, порядок asc/desc, `usageCount` исключен) и фильтр-конструктор `LogicalGroup` по колонкам name/branch/browserName/os/viewport/createdDate/markedAs (`preview`/`usageCount` без фильтра). Сортировка пишется в URL через `SearchParams`, выбранные колонки хранятся в `localStorage` (`baselinesVisibleFields`).
  - таблицы Mantine `Table` в `ScrollArea` с бесконечной прокруткой (`useInfinityScroll`, limit=20) и выбором видимых колонок через `Chip.Group`. Колонки по умолчанию: превью, name, branch, browserName, viewport, os, createdDate, usageCount, markedAs.
  - ячеек: превью 80×60 с `HoverCard` шириной 400px и плейсхолдером при отсутствии файла; usageCount как `Badge` с подсказкой; ось/браузер с иконками; даты форматируются `yyyy-MM-dd HH:mm:ss`.
  - кликабельных строк: переход на `/` с query-param `filter={"baselineSnapshotId":"<snapshootId>"}` (через `navigate`, без доп. загрузки).
  - выделения строк чекбоксами и удаления выбранных бейзлайнов через модальное подтверждение (`GenericService.delete` и refetch).
- Загрузка показывается скелетонами бесконечной прокрутки, ошибки — текстом. UsageCount информационный и не участвует в сортировке.

## Навигация и маршрутизация
- В глобальной навигации добавлен пункт “Baselines” (группа main) с переходом на `/baselines`; в Spotlight-экшенах он отображается только для ролей admin/reviewer.
- Страница использует общие хедер/навигацию `IndexLayout`, переключаясь между Tests и Baselines по pathname.
- Редирект в чек-вью использует существующую страницу `/` и меняет только query params, не трогая состояние списка тестов.

## Независимость задач
- Бэк и фронт части baselines-вью общаются через расширенную ручку `/v1/baselines` и фильтр `baselineSnapshotId` в `/v1/tests`.
- Админ-задача очистки осиротевших бейзлайнов реализована отдельной ручкой `/v1/tasks/task_handle_orphan_baselines` и не пересекается с логикой `/baselines`.
- Документация и планы вынесены в отдельные файлы, чтобы можно было реализовывать/тестировать части независимо.
